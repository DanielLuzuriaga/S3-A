Comparación entre Programación Tradicional y Programación Orientada a Objetos

La programación tradicional, también es conocida como programación estructurada, ya que se basa en la descomposición del problema en funciones que realizan tareas específicas, al desarrollar el código bajo este método, las temperaturas diarias son ingresadas mediante una función independiente y el cálculo del promedio semanal se realiza a través de otra función, lo que resulta sencillo, fácil de entender y es recomendado para problemas pequeños o de baja complejidad, ya que el flujo del programa es lineal y directo.

En cuanto a la programación orientada a objetos (POO) se organiza el código en clases y objetos, lo que nos permite modelar el problema de una forma más cercana a la realidad, ya que para este cado, cada día de clima se representa como un objeto con sus propios datos, y la semana se maneja como una entidad que agrupa estos objetos, con lo cual podemos observar el uso de principios fundamentales como el encapsulamiento, al proteger los datos internos, y la herencia, al reutilizar y extender funcionalidades entre clases.

Una diferencia clave entre ambos enfoques es la escalabilidad, ya que el código tradicional cumple correctamente su función para calcular el promedio semanal, pero si se desea ampliar puede resultar más compleja al momento de agregar nuevas características. En cambio, el código orientado a objetos facilita la extensión del sistema, nos permite incorporar nuevas variables climáticas o comportamientos sin afectar la estructura principal del programa.

En cuanto a mantenimiento y reutilización, la POO ofrece ventajas significativas, ya que los objetos pueden reutilizarse en distintos contextos y el código resulta más modular. Sin embargo, la programación tradicional presenta una curva de aprendizaje más baja y puede ser más eficiente para tareas simples.

Podemos concluir que ambos enfoques resuelven adecuadamente el problema planteado; no obstante, la programación tradicional es más apropiada para soluciones simples y directas, mientras que la programación orientada a objetos es más recomendable para sistemas más complejos, escalables y de mayor mantenimiento a largo plazo.
